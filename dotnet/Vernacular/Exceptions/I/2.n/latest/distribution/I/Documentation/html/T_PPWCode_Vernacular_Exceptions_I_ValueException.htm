PPWCode Exceptions Vernacular I
In many cases, a property exception is needed that reports the original value of the property. This value can be used to generate sensible end-user messages of the form "Unable to change {PropertyName} for {Sender} from {OldValue} to {NewValue}".

Remarks

This exception is a generalized version of a PropertyException that carries information about the OldValue and the NewValue. It is a bore to create separate exceptions for each of those specific cases. It would be nice to use generics for the type of the property value, but that is something for a later version.

This exception can be used for simple properties of all kinds: simple properties of reference type, as well as simple properties of value types, both of mutable types and immutable types. Values should be considered read-only, also if they are or reference type.

This kind of exception cannot be thrown in a constructor, since there is no original value then.
