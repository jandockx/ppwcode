PPWCode Exceptions Vernacular I

Classes

  ClassDescription
CompoundSemanticException
Vehicle for communicating more than one SemanticException at once.
Error
Exceptions that are not semantically relevant. When an exception of this type occurs, code should fail gracefully, and, in the best of cases, shut down the application, after warning the appropriate people.
ExternalError
This error is thrown when an external condition occurs, which we know can happen (however unlikely), which we do not want to deal with in our application. Exceptions that are not semantically relevant. When an exception of this type occurs, code should fail gracefully, and, in the best of cases, shut down the application, after warning the appropriate people.
ImmutableException
The method that throws this error is a mutating method, flagged not to be used, because the instance is flagged as immutable.
NoLongerSupportedError
This exception is thrown by a method defined in an API, in a later version of the API, if the method that was defined in an earlier version is no longer supported in new versions.
ProgrammingError
This error is thrown when a programming condition occurs, which we know can happen (however unlikely), which we do not want to deal with in our application. Exceptions that are not semantically relevant. When an exception of this type occurs, code should fail gracefully, and, in the best of cases, shut down the application, after warning the appropriate people.
PropertyException
PropertyExceptions are exceptions that carry with them information about the property for which they occurred. They are usually thrown by a property setter during validation.
SecurityException
Exception that signals a refusal to perform the nominal effect by a method, because of security reasons.
SemanticException
Supertype for exceptions related to semantics: the nominal effect of a method could not be reached, because doing so under the given circumstances would violate semantics (often type invariants).
ValueException
In many cases, a property exception is needed that reports the original value of the property. This value can be used to generate sensible end-user messages of the form "Unable to change {PropertyName} for {Sender} from {OldValue} to {NewValue}".
